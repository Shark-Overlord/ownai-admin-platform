# 前端开发对接指南

> 基于 2026-04-16 全量接口联调测试结果整理  
> 服务前缀：`http://localhost:8011/api`（需保持 Session / Cookie）

---

## 1. 通用约定

### 1.1 响应格式

```json
{
  "code": 0,
  "data": {},
  "message": "ok"
}
```

- `code === 0`：业务成功
- `code !== 0`：业务失败，错误信息在 `message` 中

### 1.2 全局错误码与前端处理

| 错误码 | 含义 | 前端建议处理 |
| :--- | :--- | :--- |
| `0` | 成功 | 正常渲染 |
| `40000` | 请求参数错误 | 提示用户检查输入（如空参、格式错误、重复账号） |
| `40100` | 未登录 | 跳转登录页 |
| `40101` | 无权限 | 提示“无权访问”或跳转 403 页 |
| `40400` | 请求数据不存在 | 展示空状态或 404 页 |
| `40300` | 禁止访问 | 提示禁止访问 |
| `50000` | 系统内部异常 | 提示“系统繁忙，请稍后重试” |
| `50001` | 操作失败 | 根据 message 提示（如“今日已签到”） |

### 1.3 鉴权说明

- **公开接口**：无需登录，可直接调用。
- **登录用户接口**：需先调用登录接口获取 Session Cookie，后续请求自动携带。
- **管理员接口**：仅 `userRole === 'admin'` 可访问，前端需在路由/菜单层做权限隔离。

## 2. 登录注册模块

### 2.1 注册页

#### 界面显示字段
- 账号输入框：`userAccount`
- 密码输入框：`userPassword`
- 确认密码输入框：`checkPassword`
- 注册按钮

#### 前端调用逻辑

```http
POST /user/register
Content-Type: application/json

{
  "userAccount": "testuser001",
  "userPassword": "12345678",
  "checkPassword": "12345678"
}
```

- **成功**：`data` 为新用户 ID（Long），可自动跳转登录页或执行自动登录。
- **失败常见场景**：`code=40000, message=账号已存在` → 提示账号已存在。

#### 边界
- 三字段均不能为空，后端已校验。
- 密码与确认密码一致性由后端校验（前端也建议预校验）。

### 2.2 账号密码登录页

#### 界面显示字段
- 账号输入框
- 密码输入框
- 登录按钮

#### 前端调用逻辑

```http
POST /user/login
Content-Type: application/json

{
  "userAccount": "admin",
  "userPassword": "12345678"
}
```

- **成功**：返回 `LoginUserVO`，需写入全局状态（如 Pinia / Redux）。
- 浏览器会自动写入 `JSESSIONID` Cookie，后续接口请求保持同域即可自动带 Cookie。

#### 返回字段（LoginUserVO）

| 字段 | 类型 | 前端展示位置 |
| :--- | :--- | :--- |
| `id` | Long | 用户标识（隐藏） |
| `userName` | String | 昵称 |
| `userAvatar` | String | 头像 URL |
| `userProfile` | String | 个人简介 |
| `userRole` | String | `admin` / `user`，用于权限判断 |
| `memberLevel` | String | `normal` / `plus` / `pro`，用于会员角标 |
| `pointBalance` | Integer | 积分余额 |
| `memberExpireTime` | Date | 会员到期时间（可格式化展示） |
| `createTime` | Date | 注册时间 |

#### 边界
- `code=40100`：未登录（调用受保护接口时触发）。
- 微信登录 `GET /user/login/wx_open?code=xxx` 本轮未实测，需外部微信授权环境。

### 2.3 登出

```http
POST /user/logout
```

- **成功**：`data=true`，前端需清空本地用户状态并跳转首页。

## 4. 作品浏览模块

### 4.1 作品列表页

#### 界面显示字段
- 封面：`coverUrl`
- 标题：`title`
- 简介：`summary`
- 分类名：`category.name`
- 标签列表：`tagList[].name`
- 价格：`cashPrice` / `pointsPrice`
- 会员专享角标：`memberOnly === 1`
- 浏览量：`viewCount`
- 发布时间：`createTime`

#### 前端调用逻辑

```http
POST /artwork/list/page/vo
Content-Type: application/json

{
  "current": 1,
  "pageSize": 10,
  "categoryId": 1,
  "searchText": ""
}
```

- 返回 `Page<ArtworkVO>`：
  - `records`：作品数组
  - `total`：总条数
  - `pages`：总页数

#### 边界
- `searchText` 为空字符串即可查全部。
- `size` 建议不超过 20（`user/list/page/vo` 有限制，但作品列表无硬性限制，仍建议合理分页）。

### 4.2 作品详情页

#### 界面显示字段
- 封面/视频：`coverUrl`、`videoUrl`
- 标题：`title`
- 详细描述：`description`
- 分类：`category.name`
- 标签：`tagList[].name`
- 现金价：`cashPrice`
- 积分价：`pointsPrice`
- 会员专享：`memberOnly`
- 能否查看 Prompt：`canAccessPrompt`（Boolean）
- Prompt 内容：`promptContent`（仅 `canAccessPrompt=true` 时有值）
- 不可访问原因：`accessReason`（如 `purchase_required`）
- 浏览量：`viewCount`

#### 前端调用逻辑

```http
GET /artwork/get/vo?id=2
```

- **未登录用户**：`canAccessPrompt` 对免费作品可能为 `true`；付费/会员作品通常为 `false`。
- **已购买/已开通会员用户**：`canAccessPrompt=true`，`promptContent` 有值。

#### 边界
- `id` 不存在 → `code=40400` 或 `50000`，展示作品不存在页。
- `promptContent` 为 `null` 时，前端需展示“购买后可查看”或“开通会员后可查看”。

## 5. 作品订单模块

### 5.1 创建订单（作品详情页 → 购买）

#### 界面显示字段
- 选择购买方式：现金 `cash` / 积分 `points`
- 确认购买按钮

#### 前端调用逻辑

```http
POST /order/create
Content-Type: application/json

// 现金购买
{
  "artworkId": 2,
  "orderType": "cash",
  "paymentChannel": "mock"
}

// 积分兑换
{
  "artworkId": 2,
  "orderType": "points",
  "paymentChannel": "points"
}
```

- **现金订单**：返回待支付订单（`orderStatus=pending`），需走支付流程。
- **积分订单**：后端直接扣积分并完成（`orderStatus=completed`），无需额外支付。

#### 边界
- 重复购买已拥有权限的作品 → `code=50001, message=你已经拥有该作品的访问权限`。
- 作品不存在 → `code=40400`。
- 积分不足（积分兑换时）→ 由后端校验，可能返回 `50001`。

### 5.2 模拟支付（测试环境用）

```http
POST /order/pay/mock
Content-Type: application/json

{
  "orderNo": "ORD1776306283914423269",
  "paymentChannel": "mock"
}
```

- **成功**：`data=true`，订单状态变为 `completed`。
- 支付成功后刷新作品详情，可拿到 `promptContent`。

### 5.3 我的作品订单列表

#### 界面显示字段
- 作品封面：`artworkCoverUrl`
- 作品标题：`artworkTitle`
- 订单号：`orderNo`
- 订单类型：`orderType`（`cash` / `points`）
- 订单状态：`orderStatus`（`pending` / `completed` / `cancelled`）
- 订单金额：`orderAmount`
- 积分数量：`pointsAmount`
- 支付渠道：`paymentChannel`
- 支付时间：`payTime`
- 完成时间：`finishTime`
- 创建时间：`createTime`
- 操作按钮：
  - `pending` → 显示“取消订单”
  - `completed` → 显示“查看作品”

#### 前端调用逻辑

```http
POST /order/my/list/page
Content-Type: application/json

{
  "current": 1,
  "pageSize": 10
}
```

#### 边界
- 空列表时展示“暂无订单”。

### 5.4 取消作品订单

```http
POST /order/cancel
Content-Type: application/json

{
  "orderNo": "ORDxxxxxxxx"
}
```

- 仅 `pending` 订单可取消。
- 非本人订单 → `code=40101`。

## 8. 后台管理模块（仅 admin 可见）

### 8.1 通用说明

- 所有后台接口需管理员 Cookie，否则返回 `40101`。
- 前端路由建议配置 `meta: { requiresAdmin: true }`，并在路由守卫中结合 `userRole` 判断。

### 8.2 用户管理

#### 界面显示字段
- 用户账号：`userAccount`
- 昵称：`userName`
- 角色：`userRole`
- 会员等级：`memberLevel`
- 积分余额：`pointBalance`
- 注册时间：`createTime`
- 操作：编辑、删除

#### 接口清单

| 功能 | 接口 | 说明 |
| :--- | :--- | :--- |
| 用户列表 | `POST /user/list/page` | 返回 `Page<User>`（含密码，前端不展示密码） |
| 脱敏用户列表 | `POST /user/list/page/vo` | 返回 `Page<UserVO>`（无密码） |
| 添加用户 | `POST /user/add` | 默认密码 `12345678`，MD5 加密由后端处理 |
| 查看用户 | `GET /user/get?id={id}` | 返回完整 `User` 对象 |
| 更新用户 | `POST /user/update` | 可更新昵称、积分、角色等 |
| 删除用户 | `POST /user/delete` | 逻辑删除 |

#### 边界
- `userAccount` 全局唯一，添加重复账号会返回 `50000`（逻辑删除后仍可能冲突，建议使用唯一账号）。

### 8.3 分类管理

## 10. 前端调用时序图（典型流程）

### 10.1 新用户注册 → 登录 → 浏览作品 → 购买 → 查看 Prompt

```text
1. POST /user/register        → 注册成功
2. POST /user/login           → 获取 Cookie + LoginUserVO
3. GET  /artwork/get/vo?id=2  → canAccessPrompt=false
4. POST /order/create         → 创建现金订单（pending）
5. POST /order/pay/mock       → 支付成功（completed）
6. GET  /artwork/get/vo?id=2  → canAccessPrompt=true, promptContent=xxx
```

### 10.2 开通会员流程

```text
1. POST /member/order/create  → 创建会员订单（pending）
2. POST /member/pay/mock      → 支付成功
3. GET  /user/get/login       → memberLevel 更新为 plus/pro
4. GET  /artwork/get/vo?id=3  → 会员作品 canAccessPrompt=true
```
