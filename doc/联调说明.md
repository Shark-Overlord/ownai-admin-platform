# 后端联调说明

## 1. 启动前准备

1. 执行建表脚本：`sql/create_table.sql`
2. 执行初始化脚本：`sql/init_data.sql`
3. 在 `src/main/resources/application.yml` 中配置 MySQL、COS、微信参数
4. 启动项目后访问：
   - 接口前缀：`http://localhost:8101/api`
   - Knife4j：`http://localhost:8101/api/doc.html`

## 2. 初始化账号

- 管理员账号：`admin`
- 管理员密码：`12345678`

项目使用 `md5("yupi" + password)` 作为密码摘要规则。

## 3. 推荐联调顺序

### 3.1 用户登录

接口：`POST /api/user/login`

请求体示例：

```json
{
  "userAccount": "admin",
  "userPassword": "12345678"
}
```

登录成功后保持 Session / Cookie。

### 3.2 获取基础数据

- `GET /api/category/list`
- `GET /api/tag/list`
- `POST /api/artwork/list/page/vo`

作品列表请求示例：

```json
{
  "current": 1,
  "pageSize": 10,
  "searchText": "",
  "categoryId": 1
}
```

### 3.3 查看作品详情

接口：`GET /api/artwork/get/vo?id=1`

- 免费作品会直接返回 `promptContent`
- 付费作品未购买时 `promptContent` 为空
- 会员作品未开通会员或未购买时 `promptContent` 为空

### 3.4 创建作品订单

接口：`POST /api/order/create`

现金购买示例：

```json
{
  "artworkId": 2,
  "orderType": "cash",
  "paymentChannel": "mock"
}
```

积分兑换示例：

```json
{
  "artworkId": 2,
  "orderType": "points",
  "paymentChannel": "points"
}
```

### 3.5 模拟支付成功

接口：`POST /api/order/pay/mock`

```json
{
  "orderNo": "ORDxxxxxxxx",
  "paymentChannel": "mock"
}
```

支付成功后再调用作品详情接口，可以拿到 `promptContent`。

### 3.6 取消待支付作品订单

接口：`POST /api/order/cancel`

```json
{
  "orderNo": "ORDxxxxxxxx"
}
```

管理员可调用：`POST /api/order/admin/cancel`

## 4. 会员联调

### 4.1 创建会员订单

接口：`POST /api/member/order/create`

现金开通示例：

```json
{
  "memberLevel": "plus",
  "durationDays": 30,
  "orderType": "cash",
  "paymentChannel": "mock"
}
```

积分开通示例：

```json
{
  "memberLevel": "pro",
  "durationDays": 30,
  "orderType": "points",
  "paymentChannel": "points"
}
```

### 4.2 模拟会员支付成功

接口：`POST /api/member/pay/mock`

```json
{
  "orderNo": "MEMxxxxxxxx",
  "paymentChannel": "mock"
}
```

成功后重新调用 `GET /api/user/get/login`，可以看到 `memberLevel` 和 `memberExpireTime` 更新。

### 4.3 取消待支付会员订单

接口：`POST /api/member/cancel`

```json
{
  "orderNo": "MEMxxxxxxxx"
}
```

管理员可调用：`POST /api/member/admin/cancel`

### 4.4 管理员直接发放会员

接口：`POST /api/member/grant`

```json
{
  "userId": 1,
  "memberLevel": "pro",
  "durationDays": 90,
  "description": "manual grant"
}
```

## 5. 积分联调

- 获取积分概览：`GET /api/point/me`
- 每日签到：`POST /api/point/check-in`
- 查询积分流水：`POST /api/point/record/list/page`

积分流水请求示例：

```json
{
  "current": 1,
  "pageSize": 10
}
```

## 6. 后台管理接口

- 分类：`/api/category/*`
- 标签：`/api/tag/*`
- 作品：`/api/artwork/add`、`/api/artwork/update`、`/api/artwork/delete`
- 作品订单：`/api/order/list/page`
- 会员订单：`/api/member/order/list/page`
- 操作日志：`/api/operation-log/list/page`

以上后台接口均需要管理员登录态。

## 7. 文件上传

接口：`POST /api/file/upload`

表单参数：

- `file`：文件
- `biz`：业务类型

支持的 `biz`：

- `user_avatar`
- `artwork_cover`
- `artwork_video`
- `artwork_prompt`

## 8. 常用状态值

### 作品订单 / 会员订单状态

- `pending`
- `completed`
- `cancelled`

### 作品订单类型

- `cash`
- `points`

### 会员订单类型

- `cash`
- `points`
- `admin_grant`

### 会员等级

- `normal`
- `plus`
- `pro`
